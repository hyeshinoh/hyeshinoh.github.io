MySQL Intermediate Syntax

1. 숫자 올림, 반올림, 내림

1) 올림

  • syntax: ceil(숫자)
  • 자릿수를 지정할 수 없기 때문에 아래와 같은 trick이 필요

2) 반올림

  • syntax: round(숫자, 결과 소수점 자릿수)

3) 내림

  • syntax: truncate(숫자, 결과 소수점 자릿수)
  • 참고: truncate은 table을 삭제할 때도 사용하는데, delete table과 달리 truncate table은 아예 테이블을 초기화시킴

2. 조건문

1) if

  • syntax: if(조건, 참인 경우, 거짓인 경우)

2) ifnull

  • syntax: ifnull(컬럼이름, 디폴트 데이터)

3) case when then else end

  • python의 if문과 비슷하게 사용
  • syntax: case when 조건 then 결과 else 결과 end
    • when, then은 반복 가능

3. Date Format

4. Join

  • join실습에 쓸 test1 database와 user table, addr table 생성

1) Inner Join

  • 그냥 inner join을 사용할 경우
    • user의 5개 데이터에 addr의 4개 데이터를 각각 mapping해 20 row의 데이터 생성
  • on에 매칭시킬 컬럼을 지정할 경우
    • 일치하는 데이터만 찾아서 join (두 테이블에 다 있는 데이터만 join)

2) Left Join

  • user에 있는 user_id에만 주소를 join함

3) Right Join

  • 주소만 있고 user_id, user.name이 null값인 row 생성

5. Union

  • union

  • union all: 중복이 제거되지 않음

  • union을 이용한 full outer join

    • syntax: left join 결과 union right join 결과

6. Sub-query

  • sub-query는 query문 안에 사용하는 query
  • select절, from절, where절 등에서 사용이 가능

1) select절에 사용

전체 나라수, 도시수, 언어수를 하나의 row로 출력하기

  • filtering을 먼저 한 후에 join

    • join할 때는 최대한 적은 데이터끼리 하는 것이 쿼리 성능 향상에 도움

  • join을 한 후에 filtering

3) where절에 사용

인구 800만 이상 도시의 국가코드, 국가이름, 대통령 이름을 출력

7. Index

  • 테이블에서 데이터를 검색할 때 빠르게 찾을 수 있도록 해주는 기능
  • where절에 들어가는 컬럼을 index로 설정해 놓으면 설정한 컬럼을 조건으로 검색할 때 빠르게 검색할 수 있음
  • 너무 많은 index가 설정되면 데이터가 입력될 때마다 index에 데이터를 넣어야하므로 데이터 입력시 속도가 느려질 수 있음
  • 그러므로 검색조건으로 자주 사용하는 컬럼에 설정하는 것이 좋음

1) syntax

2) example

city 테이블에 population 컬럼을 index로 추가

3) explain

query를 실행하기 전에 index로 검색을 하는지 확인 가능

4) employees db로 테스트

8. View

  • 가상의 테이블로 특정 데이터만 보고자할 때 사용 (특정 컬럼의 데이터를 보여주는 역할만 수행)
  • view를 사용함으로써 query를 더 간단하게 만들 수 있음
  • 컬럼의 수정이나 인덱스 설정 같은 명령을 할 수 없음

1) Basic

  • 일부 컬럼 조회하기

2) ALIAS

alias(as)를 이용하여 컬럼명을 변경할 수 있음
as는 생략 가능

3) DISTINCT

DISTINCT를 이용하여 특정 컬럼의 중복 데이터를 제거 가능

  • name 컬럼을 중복 제거하여 조회

4) WHERE

WHERE절을 이용해서 검색 조건을 추가
AND, OR, 연산자 등의 기능을 사용

  • 나이가 30살 이상인 user를 조회

  • 등록일이 2016-01-01일 이후의 user를 조회

  • 등록일이 2010-01-01에서 2017-12-31 사이인 user를 조회 (AND는 둘다 true이면 true, OR도 사용가능)

  • BETWEEN을 사용하여 기간 조회 (숫자데이터도 사용 가능)

5) ORDER BY

ORDER BY를 이용하여 특정 컬럼으로 데이터 정렬 가능

  • age로 오름차순 정렬 (ASC는 생략이 가능)

  • age로 내림차순 정렬

  • age로 내림차순 정렬하고 rdate를 오름차순 정렬

6) CONCAT

CONCAT을 사용하여 select한 데이터를 합쳐서 새로운 컬럼으로 보여줄 수 있음

  • name과 age를 같이 보여주도록 조회

7) LIKE

LIKE를 이용하여 특정 문자열이 들어간 데이터 조회 가능
%는 ‘어떤 문자나’와 같은 의미로 “%gmail” 은 gmail 문자열 앞에 아무 문자나 올수 있음을 의미
NOT LIKE를 사용하여 특정 문자가 들어가지 않는 데이터 조회 가능

  • email에 gmail이 들어간 데이터 조회

  • email에 gmail이 들어가지 않는 데이터 조회

8) IN

여러개의 조건을 만족하는 데이터를 조회하고 싶을때 사용
WHERE 절의 조건을 여러개 사용하는것을 간단하게 만들 수 있음

  • name이 peter와 alice인 데이터 조회

  • name이 peter와 alice인 데이터 조회

9) LIMIT

LIMIT은 조회하는 데이터의 수를 제한할 때 사용
데이터가 너무 많은 경우에는 항상 limit을 사용하여 적은 데이터를 조회해야 조회 시간이나 업데이트시의 실수를 줄일수 있음

  • user1 테이블에서 3개의 데이터 조회

  • user1 테이블에서 두번째에서 네번째까지의 3개의 데이터 조회

6. UPDATE

limit도 함께 사용해주는 것이 좋음

8. GROUP BY

GROUP BY는 여러개의 동일한 데이터를 가지는 특정 컬럼을 합쳐주는 역할을 하는 명령
sql의 그룹함수: count, min, max, sum, avg

  • world 데이터 베이스로 이동

1) GROUP BY 함수

(1) COUNT

  • city 테이블의 CountryCode를 묶고 각 코드마다 몇개의 데이터가 있는지 확인

  • countrylanguage 테이블에서 전체 언어가 몇개 있는지 구하시오.

(2) MAX

  • 대륙별 인구수와 GNP 최대 값을 조회

(3) MIN

  • 대륙별 인구수와 GNP 최소 값을 조회 (GNP와 인구수가 0이 아닌 데이터 중에서)

(4) SUM

  • 대륙별 총 인구수와 총 GNP

(5) AVG

  • 대륙별 평균 인구수와 평균 GNP 결과를 인구수로 내림차순 정렬

2) HAVING

GROUP BY에서 반환되는 결과에 조건을 줄 수 있음

  • 대륙별 전체인구를 구하고 5억이상인 대륙만 조회

  • 대륙별 평균 인구수, 평균 GNP, 1인당 GNP한 결과를 1인당 GNP가 0.01 이상인 데이터를 조회하고 1인당 GNP를 내림차순으로 정렬
